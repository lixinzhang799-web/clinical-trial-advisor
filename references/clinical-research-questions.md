# 临床研究问题点

本文件汇集临床研究设计、执行、数据、安全、质量和注册决策中的高频问题点，可直接用于解释、培训、问题分析和工作流检查。它提供的是专业判断框架，不替代现行法规、监管指南、伦理要求、研究方案、研究者手册或机构SOP。

凡涉及法定报告时限、固定阈值、表单版本、地区程序、产品特异性要求或“当前是否合规”的问题，必须再到适用监管机构或ICH官网核验。

## 1. 共同判断方法

### 1.1 先把问题变成决策题

面对“是否违规”“为什么这样设计”“应该怎么处理”时，先明确：

- 研究阶段、产品类型、适用国家或地区；
- 事件发生时间、知晓时间、首次采取措施时间和报告时间；
- 方案、知情同意书、研究者手册、安全性管理计划及SOP如何规定；
- 需要做的是概念解释、医学判断、合规判断、操作建议，还是文件起草；
- 哪些事实已经确认，哪些仍需补证。

问题中的关键事实会改变结论时，应先追问，不用假设补齐事实。

### 1.2 用一条闭环组织答案

推荐按以下链条分析：

**目标或风险 → 所需证据 → 设计或流程控制 → 决策规则 → 记录与升级 → 后续验证**

一个可靠答案至少区分：

- 已确认事实；
- 基于事实作出的专业推断；
- 尚待核验的动态要求；
- 当前可立即采取的风险控制措施。

### 1.3 区分四类来源

- 规范性来源：适用法律法规、监管要求、ICH指导原则和伦理要求；
- 研究特异来源：方案、修订案、研究者手册、知情同意书、统计分析计划和安全性计划；
- 组织控制来源：SOP、工作指引、委托协议、质量协议和系统配置；
- 方法学来源：用于解释设计逻辑、风险控制和常见实践，但不能单独证明合规。

发生冲突时，不应机械选择“级别最高”的一句话；应确认其适用范围、版本、法域和研究阶段，并记录采用依据。

### 1.4 把研究看作证据生产链

`开发目标 → 临床问题 → 方案设计 → 受试者保护 → 可执行流程 → 数据生成与沿革 → 统计分析 → 医学解释 → 研究报告 → 注册决策`

所有判断同时服务于三个质量目标：受试者得到保护、关键结果可靠、从决定到报告可追溯。迟补或纠正记录时保留真实日期、原因、来源和批准轨迹；可追溯性要求不能被解释为允许事后补造。

## 2. 试验设计与方案决策

### 2.1 从临床问题到可估计问题

设计起点不是“选择一个常见方案”，而是明确：

- 目标人群是谁，疾病阶段和既往治疗如何；
- 干预、剂量、疗程和背景治疗是什么；
- 对照代表安慰剂、标准治疗、活性对照还是历史基准；
- 主要结局在什么时间点、用什么量表或规则测量；
- 需要估计的治疗效应是什么；
- 治疗中断、救援治疗、死亡、依从性差等治疗后事件如何处理。

人群、终点、对照、分析集和缺失数据策略必须指向同一个科学问题。

### 2.2 入排标准的逻辑

每条入排标准应至少对应一种目的：保护受试者、减少无法解释的混杂、保证终点评估、满足产品风险控制，或建立目标适用人群。

判断是否过度排除时，同时检查：

- 风险是否真实且与研究干预、程序或疾病相关；
- 是否能通过加强监测、分层、稳定期要求或救援措施替代排除；
- 排除后样本是否仍代表未来用药人群；
- 标准是否可客观执行，是否依赖模糊的研究者主观判断。

例如，中重度银屑病试验排除“不稳定心血管疾病”通常是为了降低研究药物、系统性炎症、合并治疗或试验程序叠加后的急性风险，并避免近期心血管事件对安全性和疗效评价造成难以解释的干扰。具体定义应写成可操作的病史、时间窗、症状或检查标准，而不是只保留“不稳定”三个字。

### 2.3 对照、随机、盲法与偏倚

- 对照选择应服务于临床问题和伦理可接受性；
- 随机化用于平衡已知和未知预后因素，不等于自动消除所有偏倚；
- 分层因素应少而关键，并有临床或统计依据；
- 盲法应覆盖可能影响治疗、评估、数据处理和决策的角色；
- 无法盲法时，应强化独立评估、客观终点、标准化流程和盲态数据审核。

### 2.4 终点、样本量与决策规则

终点需要同时满足临床意义、测量可靠性、时间合理性和统计可分析性。样本量不是单一公式结果，应明确效应量、变异度或事件率、显著性水平、把握度、脱落、分层、期中分析和多重性假设。

非劣效设计还需要说明非劣效界值的临床与历史依据、试验敏感性、对照效应保持、依从性影响，以及ITT和PP分析不一致时的解释规则。

## 3. 早期临床与临床药理

### 3.1 共同证据链

早期临床设计应连接：非临床证据 → 既往人体信息 → 暴露预测 → 剂量与给药方案 → 采样与监测 → 递增或停止规则 → 后续开发决策。

不应脱离产品特征套用固定起始剂量、洗脱期、采样点或接受阈值。

### 3.2 BA/BE与食物影响

设计时重点判断：

- 交叉设计能否减少个体间差异；若半衰期长、存在残留效应或安全性限制，是否更适合平行设计；
- 洗脱期能否控制残留，且与代谢物和药效持续时间相符；
- 采样时程能否充分覆盖吸收峰和消除相；
- 给药前浓度、呕吐、异常给药、漏采样和伴随用药如何处理；
- 样本量是否基于合理的个体内变异和预期差异；
- 分析物、药代参数、统计模型及判定规则是否预先规定。

食物影响研究还应把餐型、进餐时间、给药时间、饮水和餐食完成度标准化。

### 3.3 首次人体试验与剂量递增

FIH的关键不是找到“一个安全起始剂量”，而是建立可逐步学习和限制风险的体系：

- 综合毒理、药理、靶点生物学、暴露和不确定性确定起始水平；
- 选择健康受试者或患者时，平衡可解释性、潜在获益和不可逆风险；
- 采用哨兵给药、分批入组和足够观察间隔；
- 每次递增前累计审阅安全性、耐受性、暴露和适用的药效信息；
- 预先规定个体、队列和研究层面的暂停、递增、降阶和终止条件；
- 新信息改变风险时，及时更新方案、研究者手册和知情信息。

### 3.4 PopPK与稀疏采样

先明确模型要支持的决策，如剂量选择、特殊人群、协变量影响或暴露—反应分析。采样设计需覆盖关键时间区间，并与实际访视可行性协调。

模型评价应同时看拟合、诊断、稳定性、预测性能和临床可解释性；协变量筛选不应只依赖统计显著性，还要看机制合理性和对剂量决策的影响。

### 3.5 DDI

DDI策略应从代谢酶、转运体、体外数据、临床药代和目标人群合并用药出发，判断是否需要临床研究、机会性评估或模型预测。临床设计应明确受试药与探针药、给药顺序、达到稳态的条件、采样、禁限用药及安全监测，并把结果转化为后续方案和说明性文件中的用药限制。

## 4. 中心、供应商与风险监查

### 4.1 启动依赖链

中心启动应满足：可行性确认 → 合同预算和职责明确 → 伦理及适用审批到位 → 人员培训和授权完成 → 系统、物资和药物就绪 → 启动放行。

“系统里显示已启动”不等于中心真正具备筛选条件，应以全部关键依赖可验证为准。

### 4.2 中心可行性与招募

可行性不能只看研究者意愿或中心自报患者量。至少核验：

- 目标患者来源、筛选漏斗和竞争试验；
- 关键检查、治疗和随访能力；
- 研究团队容量、既往表现和人员稳定性；
- 药物储存、样本处理、急救和数据录入条件；
- 伦理、合同和启动周期；
- 受试者交通、依从性、救援治疗和保留风险。

招募预测需写明基数、转化率、启动爬坡和季节性假设，并用实际数据滚动校准。

### 4.3 CRO与供应商监督

职责可以委托，申办者责任不能整体转移。选择和监督供应商时，应形成：能力与合规尽调 → 范围和RACI → 质量协议 → 交付物与KPI → 问题升级 → 变更和分包控制 → 绩效复核。

高风险接口包括安全数据库、中心实验室、影像、随机化系统、电子结局、药物供应和数据传输。接口必须明确数据所有者、频率、格式、核验、异常处理和版本控制。

### 4.4 基于风险的监查

先识别关键数据和关键流程，再把风险转化为中央、远程或现场监查策略。风险指标必须连接触发阈值、责任人、调查、纠正措施和关闭证据。

中央监查适合发现跨中心趋势、异常分布和延迟；现场活动适合核验源文件、授权、药物、知情过程和复杂流程。两者应互补，不以单一访视频率代替风险判断。

### 4.5 关闭与归档

中心关闭前确认受试者安全随访、数据和质疑、药物清点、样本、财务、必要报告和文件归档均已闭环。未解决问题应明确责任、期限和跟踪方式，不能因“关闭访视完成”而失去所有者。

## 5. CRF、数据、电子系统与随机化

### 5.1 从源数据到报告的链条

完整数据链是：源记录 → CRF或外部数据 → 清理与编码 → 对账 → 分析数据集 → TFL → 报告。每个转换都应可追溯、可解释、可复现。

### 5.2 CRF设计

只采集支持终点、安全性、方案执行和必要监管解释的数据。字段应明确单位、时间、选项、缺失原因和逻辑关系，并与方案、活动时间表、数据标准和分析计划一致。

需特别明确：

- 哪些记录是源数据，哪些由CRF转录；
- CRF版本如何与方案修订保持同步；
- 受版权保护量表是否获得许可；
- 注释CRF如何映射分析数据；
- 不必要的自由文本和重复采集是否被消除。

### 5.3 数据管理与锁库

数据管理计划应覆盖建库、测试、用户验收、录入、质疑、医学编码、外部数据、SAE对账、盲态管理、清理、锁库和解锁。

锁库不是一个按钮动作，而是质量门。锁库前至少确认：关键数据完整、质疑到达可接受状态、外部数据接收完成、编码和对账完成、重要偏离已分类、分析集所需决策已记录、授权已完成。解锁必须受控并保留原因、影响、批准和重锁证据。

### 5.4 电子系统

EDC、ePRO、IRT及其他eClinical系统的控制应贯穿选择、验证、配置、测试、上线、变更、访问、审计追踪、备份、恢复和退役。供应商验证材料不能自动替代申办者对预期用途、配置和接口的确认。

权限遵循最小必要原则；关键变更应评估对数据完整性、盲态和分析的影响。

### 5.5 随机化与紧急揭盲

随机序列生成、分层、区组、分配隐藏、药物编码和系统权限应共同保护盲态。随机化错误需评估受试者风险、用药、分析和方案偏离影响。

紧急揭盲应仅在受试者医疗处置确有需要时使用，记录原因、时间、角色和后续影响；“研究团队想知道分组”不是充分理由。

## 6. 安全性与医学监察

### 6.1 四个判断必须分开

对不良医学事件依次判断：

1. 是否属于方案规定的采集范围；
2. 严重性是否满足规定标准；
3. 严重程度属于轻、中、重哪一级；
4. 与研究干预的因果关系如何。

严重性与严重程度不是同一概念，因果关系也不决定事件是否记录。

受试者在首次用药前发生感冒，是否作为AE采集取决于方案规定的安全性采集起点和记录范围；它与研究药的因果关系通常另行判断，不能用“尚未用药”直接推出“不需要记录”。

### 6.2 安全事件处置闭环

出现严重过敏等紧急事件时，首要顺序是：立即医疗处置和保护受试者 → 获取关键事实 → 按适用路径及时通知和报告 → 持续随访至结局或稳定 → 评估对其他受试者、方案、研究者手册和知情同意的影响。

判断“治疗3天后才报伦理是否违规”不能只看“3天”：需确认事件性质、研究者或申办者首次知晓时间、当地伦理要求、方案和SOP时限，以及是否先采取了必要的紧急医疗和风险控制。若已超过适用时限，应按事实记录延迟，评估受试者保护影响，调查根因并形成CAPA；不得倒填或改写时间线。

### 6.3 医学监察

医学监察计划应明确：审阅的数据源和频率、风险信号、入组资格疑问、剂量调整、暂停规则、紧急升级、与药物警戒的接口、盲态边界及决策记录。

角色边界通常包括：

- 研究者负责现场医疗决策和受试者安全；
- 申办者医学监察负责跨中心医学一致性和研究层面风险审阅；
- 药物警戒负责个例处理、汇总分析和适用的安全报告；
- 独立委员会按章程作出独立审阅或建议。

### 6.4 趋势、信号与风险沟通

不能只逐例处理事件，还应观察发生率、严重性、时间关系、剂量关系、亚组、同类事件和跨研究趋势。新风险应触发适当的医学评估、风险最小化、文件更新、研究者沟通和必要的受试者再同意。

## 7. 试验用药、供应与资源

### 7.1 端到端供应链

试验用药链条包括需求预测、包装标签、放行、运输、接收、储存、发放、回收、清点、退回或销毁。每个节点要有责任人、条件、记录和异常处理。

### 7.2 预测与库存

供应预测至少考虑：入组速度、随机比例、剂量方案、治疗周期、中心分布、运输周期、损耗、重新供应规则、有效期和方案变更。应设置基准、加速和延迟情景，避免只用单一平均数。

### 7.3 盲态与清点

包装、标签、访问权限、运输文件、库存报告和紧急揭盲均可能破坏盲态。清点应能连接入库、发放、退回、损耗和最终处置；差异需调查并评估是否影响受试者用药、依从性、盲态或数据可信性。

### 7.4 温度偏离

标准处置顺序是：隔离受影响药物 → 保留完整温度和运输证据 → 由授权角色评估稳定性与可用性 → 批准放行或处置 → 评估已使用药物对受试者的影响 → 记录沟通和预防措施。

在评估完成前，不应凭经验继续发放，也不应擅自销毁关键证据。

### 7.5 预算与资源

预算应与活动、角色、数量和时间假设连接，覆盖中心费用、供应商、系统、药物、样本、监查、安全、统计、归档和变更储备。付款节点应与可验证交付物挂钩。资源不足时应优先保护受试者安全、关键数据和关键流程。

## 8. 质量保证、稽查与检查准备

### 8.1 QC、QA与申办者监督

- QC嵌入日常流程，确认具体交付物或步骤正确；
- QA以相对独立视角评价体系和流程是否有效；
- 申办者监督确保被委托活动持续受控。

三者互补，不能用一次稽查代替日常质量管理，也不能以“已委托CRO”消除申办者责任。

### 8.2 风险导向的质量体系

质量管理链条是：识别关键数据和流程 → 评估风险 → 设置预防和检测控制 → 监测指标和触发器 → 调查偏差 → CAPA → 有效性验证 → 经验反馈。

风险评估应关注对受试者保护和结果可靠性的影响，而不只是发生频率。

### 8.3 稽查

稽查计划结合研究重要性、阶段、复杂度、既往表现、供应商、数据趋势和重大变更。稽查证据应能回答：要求是什么、实际发生了什么、影响多大、为何发生、如何防止重复。

### 8.4 根因与CAPA

“加强培训”通常只是措施，不一定是根因。根因分析应检查流程设计、职责、工作量、系统、接口、知识、监督和组织激励。

CAPA应区分：

- 立即纠正：处理已发生问题；
- 纠正措施：消除复发原因；
- 预防措施：降低同类风险；
- 有效性检查：用预设证据确认措施真正起效。

### 8.5 持续检查准备

检查准备不是检查前集中补文件。研究期间持续维护版本、日期、签署、授权、决策理由、数据沿革、委托监督和问题闭环。禁止倒签、补造、隐藏或改写事实；迟补文件应保留真实创建日期和解释。

## 9. 项目治理与注册策略

### 9.1 统一事实底座

建立并维护Study Profile，至少包含研究目的、阶段、设计、人群、干预与对照、终点、样本量、地区、时间线、关键风险、重要文件版本和待决事项。跨职能文件应从同一事实底座更新，减少方案、IB、统计、安全、运营和注册材料间的不一致。

### 9.2 治理与关键路径

项目治理应明确决策层级、RACI、会议节奏、升级阈值、决策日志和变更控制。WBS负责拆解工作，关键路径揭示哪些依赖会影响首例入组、数据库锁定或申报。

区分：

- 风险：尚未发生的不确定事件；
- 问题：已经发生且需要处置的事件；
- 变更：经评估和授权后对基线作出的调整。

混用三者会导致责任、优先级和记录失真。

### 9.3 指标应连接行动

进度、质量和风险指标只有在连接阈值、责任人、调查、行动和关闭标准时才有管理价值。避免只汇报“红黄绿”而不说明原因和下一步决策。

### 9.4 注册策略

注册策略从目标适应证、目标人群、预期标签或说明性主张出发，形成问题—证据计划：每个关键主张需要什么非临床、临床药理、有效性、安全性、CMC或真实世界证据，何时获得，若结果不理想如何调整。

地区差异、沟通程序、申报资料结构、时限和具体技术要求可能变化，必须使用当前官方来源。监管沟通前应准备清晰问题、申办者立场、证据摘要、备选方案和期望反馈；沟通后记录决定、条件、责任和对开发计划的影响。

### 9.5 医疗器械与IVD

先确认产品分类、预期用途、风险等级、法域和监管路径，再判断需要临床试验、性能评价或其他证据。器械研究关注操作者学习曲线、设备迭代、程序一致性、盲法困难、性能终点和器械缺陷；IVD关注目标用途人群、样本谱、参照方法、分析性能与临床性能的衔接。具体分类和申报程序只能作为检索线索，须用当前官方来源核验。

## 10. 文件闭环与终局质量门

### 10.1 跨文件一致性

重点核对：

| 上游变化 | 必查下游 |
|---|---|
| 研究目标或终点 | 方案、SoA、CRF、SAP、样本量、TFL、CSR |
| 入排标准 | 招募、筛选、CRF、监查、医学监察、可行性 |
| 剂量或给药 | IB、ICF、供应、IRT、清点、安全监测、SAP |
| 安全风险 | IB/RSI、方案、ICF、安全计划、培训、监查、DSUR/CSR |
| 访视或程序 | SoA、ICF、CRF、预算、供应、实验室手册、系统配置 |
| 随机化或盲法 | IRT、药物编码、权限、紧急揭盲、数据审核、SAP |
| 外部供应商或数据源 | 合同、质量协议、接口、传输、核验、归档 |
| 方案修订 | 伦理/监管、版本切换、培训、CRF/EDC、SAP、供应和已入组受试者 |
| 数据处理规则 | DMP、SAP、数据集、TFL、CSR和可追溯记录 |

### 10.2 回答或交付前的六个质量门

1. 问题是否已具体到可以作出决定；
2. 事实、推断和未知是否已分开；
3. 结论是否与适用来源层级相匹配；
4. 是否说明了对受试者安全和数据可靠性的影响；
5. 是否给出可执行的下一步、责任人和证据；
6. 动态监管要求是否已通过官网核验。

### 10.3 不确定时的安全表达

如果无法可靠确认，不应用“通常如此”“大概率合规”等模糊表述替代结论。应明确指出缺少的事实或依据，并提供：

- 适用监管机构或ICH官网入口；
- 建议检索的文件类型和关键词；
- 需要核对的版本、生效日期、适用范围和具体条款；
- 在核验完成前可采取的保守风险控制措施。

本《临床研究问题点》可作为分析与工作底稿直接使用；涉及规范性结论时，应把最终依据落到当前有效的官方文件和研究特异文件上。
