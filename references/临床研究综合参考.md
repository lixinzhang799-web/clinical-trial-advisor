## 0. 使用契约

### 0.1 本文件能做什么

用于支持以下任务：

- 解释临床试验设计、执行、数据、安全、质量、运营和报告的通用逻辑；
- 把模糊问题转化为决策题、资料缺口、操作步骤、风险控制和质量门；
- 审查方案、CRF、监查计划、数据流程、供应链、CSR 等是否形成上下游闭环；
- 为临床团队培训、SOP/计划框架、项目复盘和方法学 QC 提供检查维度。

### 0.2 本文件不能替代什么

不得单独用于确认：

- 当前有效的法律法规、监管指南、伦理要求及其实施状态；
- 法定或监管报告时限、固定阈值、表单和数据库版本；
- 中国、美国、欧盟、日本等地区的当前申报路径和程序；
- 特定试验的方案、研究者手册、知情同意书、SAP、安全计划、SOP 和合同要求；
- 产品特异性剂量、洗脱期、采样点、样本量、非劣效界值或安全结论。

遇到上述问题，Skill 必须检索并打开适用监管机构或 ICH 的当前官方原文，核对标题、版本、状态、发布日期、生效日期、适用范围和支持结论的正文位置。
涉及伦理或法定报告时限时，还必须同时核对适用伦理要求、方案及 SOP 的当前版本。

### 0.3 来源优先级

按以下顺序分配来源角色：

1. 适用法域的现行法律法规和监管要求；
2. 已实施的 ICH 及正式官方指导原则；
3. 研究特异文件：方案、修订案、IB、ICF、SAP、安全计划等；
4. 组织控制文件：SOP、工作指引、委托协议、质量协议和系统配置；
5. 本综合参考：用于解释、串联、发现风险和形成行动框架。

本文件不能把“行业实践”改写成“法规强制要求”。

来源之间出现冲突时，不机械选择“层级最高”的一句话；先核对适用范围、版本、法域、研究阶段和项目约束，并记录最终采用依据。

## 1. 总纲：临床试验是一条证据生产链

临床试验不只是中心执行或数据收集，而是一条相互依赖的证据生产链：

`开发目标 → 临床问题 → 方案设计 → 受试者保护 → 可执行流程 → 数据生成与沿革 → 统计分析 → 医学解释 → 研究报告 → 注册决策`

任何局部决定都要追踪上下游影响。例如，终点变化不仅修改方案，还可能影响活动时间表、CRF、随机化或盲法、数据核查、SAP、样本量、TFL、CSR 和受试者负担。

### 1.1 共同判断闭环

对任何临床试验问题，按以下顺序处理：

1. **定义决定**：用户究竟要解释、选择、执行、纠偏、审查还是写文件；
2. **建立问题画像**：产品、适应证、阶段、设计、法域、日期、角色、现有文件和数据状态；
3. **识别关键目标或风险**：受试者保护、结果可靠性、注册用途、运营可行性；
4. **确定所需证据**：哪些事实已确认，哪些必须由官方或项目文件补证；
5. **设计控制**：预防、检测、升级和记录机制；
6. **规定决策规则**：谁在什么条件下作出继续、暂停、修改或关闭决定；
7. **形成闭环**：责任人、时间点、证据、批准、后续验证和跨文件更新。

关键事实会改变结论时，先澄清再判断，不用假设补齐。任何风险、进度或质量指标都必须连接触发阈值、责任人、调查、行动和关闭证据。

具体答复统一采用第13节的最小回答模板；简单问题可压缩，但不省略会改变结论的边界、缺口和停止条件。

### 1.2 三个根本质量目标

- **受试者保护**：权利、安全和福祉优先；
- **结果可靠性**：关键数据和关键流程足以支持可信结论；
- **过程可再现**：从决定、执行、记录到报告均可追溯，能够解释“谁、何时、基于什么、做了什么、为何这样做”。

实现这些目标的前提是试验科学且伦理可接受、职责清楚、人员合格、资源充分。

“没有记录就不能证明发生过”应理解为可追溯性要求，而不是鼓励事后补造记录。迟补、纠正或重建文件时，必须保留真实日期、原因、来源和批准轨迹。

## 2. 伦理、GCP 与角色责任

### 2.1 伦理判断顺序

`科学问题是否必要 → 风险是否合理并已最小化 → 受益风险是否可接受 → 选择受试者是否公平 → 知情同意是否真实有效 → 隐私与数据保护是否充分 → 持续审查是否到位`

科学性本身是伦理要求的一部分：无法回答问题、样本量明显不足或流程无法执行的研究，会让受试者承担没有合理知识收益的负担。

### 2.2 知情同意不是签字动作

知情同意是贯穿研究的持续过程，不是一次签字动作。有效知情同意至少包含：适当信息、可理解表达、充分时间、自主决定、提问机会、无不当影响、正确版本、过程记录和持续更新。

新风险、程序变化或可能影响继续参加意愿的信息出现时，应评估是否需要更新知情材料和重新同意。特殊人群需额外评估同意能力、法定代理、受试者本人意愿、最小风险和直接受益。

### 2.3 职责可以委托，责任不能消失

- 研究者/机构：现场医疗决定、资格确认、知情过程、方案执行、授权监督、源数据、试验用药、及时报告和受试者安全；
- 申办者：研究设计、资源、质量体系、安全监督、供应商监督、跨中心风险、监管沟通和证据完整性；
- CRO/供应商：按合同和质量协议完成被委托活动，保留可验证证据；
- 监查员：验证受试者保护、方案执行、关键数据和问题闭环；
- 伦理委员会：独立审查风险获益、受试者保护、同意材料、招募与补偿、变更及安全信息；
- QA：独立于被评价活动，通过稽查或体系评价识别系统性风险，并向适当管理层报告。

具体委托与监督控制统一按第5.3节落实，避免在角色原则和执行流程中重复列项。

## 3. 从开发目标到方案

### 3.1 设计起点

先明确目标产品特征和临床开发计划，再把单项试验放入整体证据链。每个试验应回答一个可表述的临床问题：

- 目标人群是谁；
- 干预、剂量、疗程和背景治疗是什么；
- 比较对象是什么以及为何伦理和科学上可接受；
- 结局变量在何时、以何种工具和规则测量；
- 希望估计或判断的治疗效应是什么；
- 中断治疗、救援治疗、死亡、依从性差等事件如何处理；
- 结果将支持什么下一步决定。

人群、终点、对照、分析集和缺失数据策略必须指向同一个科学问题。

### 3.2 设计选择

对平行、交叉、单臂、随机、开放、盲法、优效、非劣效等设计，不按“常用模板”选择，应比较：

- 与临床问题的匹配度；
- 偏倚控制能力；
- 伦理可接受性；
- 受试者负担；
- 样本量与时间；
- 运营可行性；
- 结果可解释性和注册用途。

随机化用于平衡已知和未知预后因素，但不能自动消除选择、实施、测量、失访或报告偏倚。无法盲法时，应加强独立评估、客观终点、标准化操作和盲态数据审核。

### 3.3 入排标准

每条标准至少服务于一种目的：保护受试者、减少难以解释的混杂、保证终点评估、落实产品风险控制或界定目标人群。

审查问题：

- 是否可以通过加强监测、稳定期、分层或救援措施代替直接排除；
- 是否把未来真实用药人群排除得过多；
- 是否可客观执行，包含明确病史、时间窗、检查或阈值；
- 是否与中心实际患者来源和招募能力相容；
- 是否会造成不公平选择或不必要负担。

例如，中重度银屑病试验排除“不稳定心血管疾病”通常用于降低研究药物、系统性炎症、合并治疗或试验程序叠加后的急性风险，并减少近期心血管事件对安全性和疗效解释的干扰。具体标准应转化为可操作的病史、时间窗、症状或检查要求，不能只保留“不稳定”这一主观表述。

### 3.4 终点和测量

终点同时满足：临床意义、测量可靠性、时间合理性、可执行性和统计可分析性。主要、次要、探索性终点的层级必须与研究目标、多重性和报告主张一致。

使用复合终点、替代终点、PRO、生物标志物或全局评价时，应明确组成、定义、事件判定、测量工具、缺失处理和临床解释。

### 3.5 样本量

样本量至少需要：研究假设、结局类型、效应量、变异度或事件率、显著性水平、把握度、分配比、多重性、期中分析、失访/不可评价比例和分析方法。

缺关键参数时，Skill 只能输出计算框架、情景范围和资料缺口，不能制造精确人数。

非劣效研究还需检查：界值的临床与历史依据、对照效应保持、试验敏感性、依从性、交叉、救援治疗，以及不同分析集结论不一致时的解释。

### 3.6 方案形成与修订

方案不是单人写作产物。推荐流程：

`开发策略/概念 → 方案纲要 → 跨职能输入 → 设计决策日志 → 可行性验证 → 完整方案 → 下游文件同步 → 培训与放行`

修订时记录变更理由和批准，并按第11节“方案修订”行统一评估受影响对象和下游更新。

## 4. 早期临床与临床药理

共同证据链：

`非临床证据 → 既往人体信息 → 暴露预测 → 剂量/给药 → 采样与监测 → 递增/停止规则 → 下一开发决定`

### 4.1 BA/BE

根据半衰期、残留效应、个体内变异和安全性选择交叉或平行设计；洗脱期应控制原型药、代谢物和药效残留；采样应覆盖吸收峰和消除相；样本量应基于合理的个体内变异和预期差异；预先规定给药前浓度、呕吐、漏采样、异常给药、伴随用药、分析物、药代参数、统计模型、判定标准和不可评价规则。食物影响研究还应标准化餐型、进餐时间、给药时间、饮水和餐食完成度。特殊制剂、长半衰期、高变异、窄治疗指数或内源性药物不能机械套用常规设计。

### 4.2 首次人体和剂量递增

起始剂量不能脱离毒理、药理、靶点生物学、暴露预测和不确定性。选择健康受试者或患者时，应平衡可解释性、潜在获益和不可逆风险。风险控制可包括哨兵给药、分批入组、观察间隔和累计审阅；每次递增前累计审阅安全性、耐受性、暴露和适用的药效信息。必须预先规定个体、队列和研究层面的暂停、降阶、递增和终止标准；新信息改变风险时同步评估方案、IB和知情材料更新。

### 4.3 PopPK

先定义模型要支持的决定，再设计采样。稀疏采样需覆盖关键时间区间并与实际访视可行性协调。模型评价不仅看拟合或统计显著性，还看稳定性、预测性能、机制合理性、协变量的临床影响和对剂量建议的可解释性。

### 4.4 DDI

从代谢酶、转运体、体外信息、临床药代和目标人群合并用药出发，决定临床研究、机会性评估或模型预测。临床设计明确受试药与探针药、给药顺序、稳态条件、采样、禁限用药和安全监测。结果必须转化为后续试验的禁限用药、剂量调整、监测和说明性文件内容。

## 5. 启动、中心、CRO、招募和监查

### 5.1 启动质量门

`可行性确认 → 职责、合同与预算明确 → 伦理/监管条件满足 → 团队授权和培训完成 → 系统、物资、药物和实验室就绪 → 书面放行`

系统中标记“已启动”不等于中心具备安全筛选和入组条件。

### 5.2 中心可行性

不能只采信研究者意愿或自报患者量。核验患者来源、筛选漏斗、竞争试验、关键检查、治疗和救援能力、团队容量、人员稳定性、既往质量、急救条件、数据录入、药物和样本条件、伦理合同周期以及受试者交通和随访负担。

招募预测应写明患者基数、转化率、启动爬坡、季节性和中心差异，并用实际数据滚动校准。

### 5.3 CRO 与供应商

供应商监督形成：`能力与合规尽调 → 范围和RACI → 质量协议 → 接口与升级规则 → 交付物与KPI → 分包/变更控制 → 绩效复核与监督证据`。

高风险接口包括安全数据库、中心实验室、影像、IRT、电子结局、药物供应和外部数据传输。每个接口明确数据所有者、频率、格式、版本、核验、异常处理和升级。

### 5.4 基于风险的监查

先识别关键数据和关键流程，再配置中央、远程和现场活动；风险指标按第1.1节的通用规则形成调查和关闭闭环。

- 中央监查：跨中心趋势、异常分布、延迟和一致性；
- 现场监查：知情过程、源文件、授权、药物、复杂流程和现场实际状态。

固定访视频率或100%源数据核查不能代替风险判断。

### 5.5 招募、留置和依从性

招募材料和渠道必须经过适当审查，不得夸大获益或弱化风险。留置策略应减少可避免负担，改善沟通、交通、访视安排和治疗连续性，而不是对受试者形成不当影响。

失访或撤回时记录联系、判断和处置过程，区分撤回干预、撤回随访、撤回数据使用等不同情形，按适用文件处理。

### 5.6 中心关闭

关闭前确认安全随访、数据质疑、药物清点、样本、财务、必要报告和文件归档均已闭环。未解决事项必须保留责任人、期限和跟踪机制。

## 6. 文档、CRF、数据和电子系统

### 6.1 文档质量

必需记录和数据应能重建“做了什么、为什么、由谁、何时、结果如何”，并具备可归属/可识别、清晰、同步、原始/真实、准确、完整、一致、持久和可获得等属性。作者、版本、元数据、审核、批准、生效、培训、分发、真实创建/归档时间、位置、访问、留存和处置均应受控。迟补文件保留真实创建日期、原因和解释；禁止倒签、补造、隐藏或改写事实。

eTMF/主文档不是项目结束时集中上传的仓库，而是持续证明试验如何被设计、批准、执行、监督和关闭的证据系统。

### 6.2 CRF

只采集支持终点、安全性、方案执行和必要解释的数据。字段需明确单位、时间、选项、缺失原因和逻辑关系，并与方案、SoA、数据标准和 SAP 一致。

审查：

- 源数据位置是否定义；
- 版本是否与方案修订同步；
- 注释 CRF 是否支持数据集映射；
- 量表许可是否到位；
- 是否存在重复采集、模糊选项和不必要自由文本；
- 填写指南是否明确更正、缺失、日期、访视和早退规则。

### 6.3 数据链

`源记录 → CRF/外部数据 → 清理和编码 → 对账 → 分析数据集 → TFL → CSR`

每次转换都应可追溯、可解释、可复现。数据更正不得覆盖原值或破坏审计追踪。

### 6.4 数据管理与锁库

数据管理计划覆盖建库、验证、用户验收、录入、质疑、编码、外部数据、SAE 对账、盲态、清理、锁库和解锁。

锁库前至少确认：关键数据完整、质疑状态可接受、外部数据齐备、编码与对账完成、重要偏离已分类、分析集所需决定已记录、授权完成。解锁必须记录原因、影响、批准、更正和重锁。

### 6.5 电子系统

EDC、ePRO、IRT 等系统应按预定用途和风险控制其选择、验证、配置、测试、上线、接口、运行、变更和退役。访问权限、角色分离、审计追踪、备份、恢复、安全和培训均须与风险相称。供应商的通用验证不能替代申办者对本研究配置和接口的确认。

权限遵循最小必要原则；关键变更评估数据完整性、盲态和分析影响。

### 6.6 随机化和揭盲

随机方法、分层、区组、分配隐藏、药物编码和权限共同保护分配过程。分层因素应少而关键。
随机化操作、系统权限、数据质疑、传输、编程和审核全过程均应保护盲态。

紧急揭盲只为受试者医疗处置确有需要时使用，记录原因、时间、角色和后续影响。随机化错误须评估用药、安全、盲态、偏离和分析影响。

## 7. 安全性、药物警戒和医学监察

### 7.1 先分开六个判断

1. 是否在方案规定的采集范围内；
2. 是否满足严重性标准；
3. 严重程度为轻、中或重；
4. 与研究干预的因果关系如何；
5. 是否符合适用RSI下的预期性判断；
6. 应进入何种记录、通知、快速报告、累计评价或周期报告路径。

严重性不等于严重程度，因果关系也不决定事件是否记录。预期性和报告路径还依赖RSI版本、法域、角色及知晓时间。

例如，首次用药前发生的感冒是否作为AE采集，取决于方案规定的安全性采集起点和记录范围；它与研究药的因果关系应另行判断，不能用“尚未用药”直接推出“不需要记录”。

### 7.2 个例处置闭环

`立即医疗处置 → 获取关键事实 → 分类与医学评估 → 按适用路径报告 → 持续随访 → 对其他受试者和研究整体的影响评估 → 文件/沟通/风险控制更新`

个例医学评估同时检查时间关系、替代原因、合并治疗、去挑战/再挑战、结局和随访完整性。

判断是否逾期不能只看事件到报告相隔几天；需确认事件性质、首次知晓时间、适用法域、伦理要求、方案和 SOP。不得倒填或改写时间线；确认逾期时应如实记录延迟，评价受试者保护影响，调查根因并采取适当CAPA。

### 7.3 从个例到信号

安全工作分为个例处置、汇总评价和研究/开发决策三层。个例处理不等于风险评价；汇总分析应使用可解释的暴露分母，并检查发生率、严重性、时间关系、剂量关系、亚组、同类事件、跨中心和跨研究趋势以及编码一致性，形成：

`新信息 → 与既往认识比较 → 风险是否改变 → 对受试者/试验/开发计划的影响 → 行动与验证`

行动可包括加强监测、更新方案/IB/ICF/监查措施、暂停或调整研究及监管沟通；每项行动保留责任、触发条件和有效性验证。

### 7.4 医学监察

医学监察计划明确审阅数据源和频率、医学风险触发器、资格疑问、剂量调整、暂停规则、紧急升级、与药物警戒接口、盲态边界和决策记录。

研究者负责现场医疗决定；申办者医学监察负责跨中心医学一致性和研究层面风险；药物警戒负责个例和汇总安全流程；独立委员会按章程独立审阅或建议。

### 7.5 上市后风险管理

把自发报告、研究数据、文献、监管信息和其他来源放入统一信号流程：检测、验证、优先级、评估、决策、风险最小化、沟通和有效性评价。不能用报告数量直接代表发生率，也不能用数据缺失推出“未发现风险”。

## 8. 试验用药、样本、供应和预算

### 8.1 端到端供应链

`需求预测 → 包装标签 → 放行 → 运输 → 接收 → 储存 → 发放 → 回收 → 清点 → 退回/销毁`

每个节点明确责任、条件、记录和偏差处置。预测同时考虑入组速度、随机比例、剂量、疗程、中心分布、运输周期、损耗、有效期、再供应规则和方案变更，并设置基准、加速、延迟情景。

### 8.2 盲态与清点

包装、标签、访问权限、运输文件、库存报告和紧急揭盲均可能破坏盲态。清点应连接入库、发放、退回、损耗和最终处置；差异需调查并评估是否影响受试者用药、依从性、盲态或数据可信性。

### 8.3 温度偏离

`隔离 → 保留温度和运输证据 → 授权角色评估稳定性/可用性 → 批准放行或处置 → 评估已使用药物对受试者影响 → 记录与预防`

评估完成前不凭经验继续发放，也不擅自销毁关键证据。

### 8.4 生物样本

采集、标签、时间、处理、离心、分装、储存、运输、接收、检测和销毁应形成可追溯链。样本身份、温度、时间偏差和链条断点须评估对受试者安全、终点和可用性的影响。

### 8.5 预算与付款

预算连接活动、角色、数量和时间假设，覆盖中心、供应商、系统、药物、样本、监查、安全、统计、归档和变更储备。付款节点与可验证交付物挂钩。资源冲突时优先保护受试者安全、关键数据和关键流程。

## 9. QA、QC、稽查和检查准备

- QC：嵌入日常流程，确认具体步骤或交付物正确；
- QA：相对独立地评价体系和流程是否有效；
- 申办者监督：确保委托活动持续受控。

偏差先评价对受试者安全/权益、主要终点、随机化和盲态、数据可靠性及监管义务的影响，再决定分类和升级。记录事实、范围、时间、即时纠正、根因、CAPA、有效性以及对分析和报告的影响。

风险质量链：

`关键数据/流程 → 失败模式 → 风险 → 预防和检测控制 → 指标/触发器 → 调查 → CAPA → 有效性检查 → 经验反馈`

风险评价以对受试者保护和结果可靠性的影响为主，不只按发生频率排序。

### 9.1 稽查

稽查计划结合研究重要性、阶段、复杂度、既往表现、供应商、数据趋势和重大变更。稽查证据应回答：要求是什么、实际发生了什么、影响多大、为何发生以及如何防止重复。

### 9.2 根因和 CAPA

“加强培训”通常只是措施，不必然是根因。根因分析应检查流程设计、职责、工作量、系统、接口、知识、监督和组织激励。

- 立即纠正：处理已发生的问题；
- 纠正措施：消除复发原因；
- 预防措施：降低同类风险；
- 有效性检查：以预先规定的证据确认措施起效。

### 9.3 持续检查准备

检查准备贯穿研究全程。按第6.1节的必需记录控制，持续保留版本、日期、签署、授权、决定理由、数据沿革、委托监督和问题闭环证据，使每项关键活动可重建、可解释。

## 10. 项目治理、CSR 和注册证据

### 10.1 统一事实底座

建立 Study Profile，至少包含：研究目的、阶段、设计、人群、干预与对照、终点、样本量、地区、时间线、关键风险、主要文件版本、重要决定和待决事项。

方案、IB、ICF、SAP、安全文件、运营计划、TFL、CSR 和注册材料从同一事实底座更新。

### 10.2 治理

明确决策层级、RACI、会议节奏、升级阈值、决定日志和变更控制。

- 风险：尚未发生的不确定事件；
- 问题：已经发生并需处置的事件；
- 变更：经评估和批准后对基线的调整。

WBS用于拆解工作，关键路径用于识别会影响首例入组、数据库锁定或申报的上游依赖；两者不能被单一状态汇报替代。

### 10.3 CSR

CSR 建立在以下一致性链上：

`方案/修订 → SAP → 数据审核决定 → 数据库 → 分析数据集 → TFL → 正文 → 附录`

写作前锁定数据截止、文件版本、分析集、盲态审核决定和主要输出。报告不是把表格转写成句子，而是解释研究为何这样设计、实际如何执行、发生了哪些偏离、结果能回答什么、不能回答什么，以及获益风险如何理解。

### 10.4 注册策略

从目标适应证、目标人群和预期标签/主张倒推问题—证据计划：每个关键主张需要哪些非临床、临床药理、有效性、安全性、CMC 或其他证据，何时获得，结果不理想时如何调整。

地区程序、时限和技术要求属于动态信息，必须单独进行当前官方核验。

监管沟通前准备清晰问题、申办者立场、证据摘要、备选方案和期望反馈；沟通后记录决定、条件、责任以及对开发计划和文件的影响。

### 10.5 医疗器械和 IVD

先确认产品分类、预期用途、风险等级、法域和适用监管路径，再确定是否需要临床试验、性能评价或其他证据。器械研究特别关注操作者学习曲线、设备迭代、程序一致性、盲法困难、性能终点和器械缺陷；IVD 关注目标用途人群、样本谱、参照方法、分析性能与临床性能的衔接。

其中涉及的具体分类、申报路径和地区程序只能作为检索线索，不能直接作为当前结论。

## 13. 最小回答模板

当 Skill 使用本文件回答专业问题时，优先采用：

1. **结论**：一句话回答当前能否判断；
2. **适用边界**：产品、阶段、法域、日期和假设；
3. **方法学判断**：目标/风险—证据—控制—决策规则；
4. **上下游影响**：受试者、方案、数据、安全、运营和报告；
5. **立即行动**：按优先级给出具体动作；
6. **资料缺口**：缺什么、由谁提供、影响什么结论；
7. **官方核验**：动态要求的官网、检索词和核对字段；
8. **下一质量门**：满足什么条件后才能继续。

本文件的核心不是记忆所有细节，而是让 Skill 始终沿着“临床问题—证据—设计—执行—数据—解释—行动”的闭环工作。
