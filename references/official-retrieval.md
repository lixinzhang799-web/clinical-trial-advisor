# ICH、NMPA与CDE官方检索流程

## 何时启动

满足任一条件即启动，不必等待用户再次授权：

- 本地蒸馏资料没有直接覆盖该问题；
- 用户要求“查一下、检索、找指导原则、按官网回答”；
- 问题涉及最新版本、实施状态、递交期限、文号或中国现行要求；
- 需要适应证、产品类型或特殊人群的专门指导原则；
- 本地资料与已知新规可能冲突。

如果法域、产品或研发阶段会改变检索结果，先问一个高信息量问题；否则基于明确假设直接检索。

## 第一步：把问题转成检索对象

提取：`法域 + 产品 + 阶段 + 主题 + 文件类型 + 时间点`。

示例：

- “中国、创新药、确证性试验、患者报告结局、正式技术指导原则、当前”；
- “ICH、药物、研发期安全、DSUR、Step 4指导原则、当前”；
- “中国、细胞治疗、长期随访、正式指导原则、2026年”。

## 第二步：先查本地，再查官网

1. 用 `scripts/search_refs.py` 搜索主题词、同义词和英文缩写。
2. 若本地结论足够且不易变化，可直接回答并引用本地原始来源章节。
3. 若覆盖不足或内容易变化，继续官网检索。

## 第三步：官方站点优先级

### ICH

1. `ich.org` 主题页用于确认指导原则家族与当前状态；
2. `database.ich.org` 或 `admin.ich.org` 的原始PDF/记录用于正文和版本；
3. 核对 Step 2/3/4、最终版/草案、发布日期、勘误和附件；
4. 地区是否实施 ICH 文件，另查该地区监管机构，不能从 ICH Step 4 自动推定当地法律效力。

搜索式示例：

- `site:ich.org <主题> guideline`
- `site:database.ich.org <编号> Step4 Final Guideline PDF`
- `site:admin.ich.org <编号> guideline`

### 中国NMPA/CDE

1. `nmpa.gov.cn`：法律法规、部门规章、公告、GCP及实施文件；
2. `cde.org.cn`：药品技术指导原则、ICH工作动态、征求意见、问答和审评相关文件；
3. 用标题、文号、发布日期和附件交叉确认；
4. 明确区分“正式发布”“征求意见稿”“问答/培训材料”“新闻或解读”；
5. NMPA/CDE原站页面不可访问时，可用政府网站转载页定位，但须继续尝试原文附件，并在答案中说明证据层级。

搜索式示例：

- `site:nmpa.gov.cn <主题> 公告 指导原则`
- `site:cde.org.cn <主题> 技术指导原则`
- `site:cde.org.cn <准确标题或ICH编号>`

## 第四步：候选文件筛选

对每个候选记录以下字段：

| 字段 | 要求 |
|---|---|
| 标题 | 使用官方完整标题 |
| 发布机构 | ICH、NMPA、CDE或其他适用机构 |
| 编号/文号 | 有则记录，不推测 |
| 版本/步骤 | R版本、Step、正式/草案/勘误 |
| 发布与实施日期 | 分开记录；没有实施日期则写未说明 |
| 适用范围 | 产品、阶段、人群、问题 |
| 相关定位 | 章节、条款、表格或附件 |
| 原始链接 | 直接指向官方页面或官方附件 |

排除仅标题相似但法域、产品、阶段或状态不适用的文件。
不要根据文件简称或标题自行扩大适用范围；必须阅读“目的、范围、范围外事项”章节。例如涉及真实世界外部对照时，不能因文件讨论RWD就自动认定其直接适用。

## 第五步：阅读与证据提取

- 打开原页面和附件，搜索关键术语并阅读上下文；
- PDF中的表格、流程图、脚注或版式会影响理解时，查看相应页面图像；
- 将规范性用词保留为准确层级：法规中的“应当/必须/不得”与指导原则中的“建议/可考虑”不可互换；
- 逐字引用前核对原文；一般优先准确释义，避免长段复制；
- 至少建立 `结论—来源—章节—适用条件` 四项映射。

## 第六步：冲突处理

1. 先比较法域和适用日期；
2. 再比较法律/监管层级；
3. 再比较正式版、草案和历史版本；
4. 再比较通用与治疗/产品特异性文件；
5. 无法消解时并列说明，不自行宣布某文件失效。

典型组合：ICH提供通用方法学，NMPA法规决定中国法律义务，CDE治疗领域指导原则补充具体设计。三者可以同时适用。

## 第七步：回答模板

按以下顺序输出：

1. **直接结论**：回答“有没有、适不适用、当前是什么状态、应怎么做”；
2. **适用边界**：法域、产品、阶段、日期和假设；
3. **官方依据**：文件名、状态、章节、官方链接；
4. **方法学解释**：与本地蒸馏知识衔接；
5. **行动建议**：下一步文件、数据或决策；
6. **检索说明**：检索日期、搜索机构、未找到或未核实的部分。

## 失败与降级

- 官网暂时不可访问：说明无法完成当前核验；可给出本地知识的暂定答案并标记可能过时。
- 只找到征求意见稿：明确其不是正式要求，不写成已实施。
- 只找到二手转载：说明来源层级并继续寻找官方原文；找不到时不要声称“根据NMPA原文”。
- 没有专门指导原则：说明检索范围和未发现结果，再用最接近的通用指导原则回答；不要编造一个文件。
