# 法规与指导原则工作参考

本文件将临床研发常用的 ICH、NMPA 和 CDE 文件整理为可检索的法规地图、核心判断规则和官方核验流程。它不保存法规全文，也不以静态内容确认当前版本、实施状态、时限或程序。

本文件与其他reference的分工：

- 本文件回答“依据什么、文件之间是什么关系、应核验哪个版本、规范性要求如何落地”，并包含DSUR与CSR的资料准备、编写、对账及QC；
- `design-statistics.md`回答设计和统计方法如何选择；
- `gcp-safety-operations.md`回答GCP、安全和运营问题如何执行。

## 1. 使用契约与来源层级

### 1.1 本文件可以做什么

- 根据问题定位应读取或检索的法规、指导原则及章节；
- 解释不同文件在安全、设计、统计、GCP、报告和申报中的职责；
- 提取不依赖时效性的核心方法学原则；
- 建立`结论—依据—正文定位—适用条件—行动`证据链；
- 识别本地文件的历史版本、示例、翻译件和当前正式文件之间的差异。

### 1.2 本文件不能替代什么

下列事项必须联网打开官方原文核验：

- 当前有效版本、ICH Step状态、中国实施状态和过渡期；
- 法定或监管报告时限、递交入口、表单、系统和数据标准；
- 文件是否正式发布、待生效、征求意见、替代、废止或撤回；
- 产品、适应证、人群、法域或活动日期特异的监管要求；
- 对外、伦理、监管沟通、检查或提交级结论。

### 1.3 来源层级

按适用性而非文件知名度使用证据：

1. 适用法域现行法律法规及具有约束力的监管文件；
2. 已在该法域实施的ICH指导原则；
3. 监管机构正式发布的通用、产品或治疗领域技术指导原则；
4. 项目批准文件、方案、IB、ICF、SAP、安全计划和正式监管沟通；
5. 征求意见稿、问答、培训或示例，仅作辅助并标明状态；
6. 方法学文献和经核验的实践知识。

ICH达到Step 4不自动等于在中国或其他地区已经实施。翻译件便于阅读，但涉及关键措辞时同时核对ICH官方原文及中国实施文件。

## 3. SUSAR、RSI与DSUR监管链

将安全信息作为一条闭环处理：

`个例SAE → 因果性 → 预期性/RSI版本 → SUSAR快速报告 → 类似事件汇总 → 信号 → 风险表征 → 文件和措施更新 → DSUR累计评价`

### 3.1 RSI的核心用途

RSI通常位于IB中，用于判断可疑严重不良反应的预期性。它不是试验药全部安全性信息的汇总，也不能为了减少SUSAR报告而扩大范围。

形成RSI时检查：

- 事件是否已在本试验药中观察到；仅根据药理机制或同类药推测，不足以列为预期严重不良反应；
- 是否有充分医学证据支持与试验药的合理因果关系；
- 单个事件通常不足以纳入，除非存在有力证据；重复发生也不自动等于应纳入；
- 对无法评价相关性的SAE，按适用要求谨慎处理；“无法评价”本身不足以把该事件加入RSI；
- 尚未上市试验药的致死事件通常不应作为预期严重不良反应写入RSI；危及生命或致死情形须特别谨慎；
- 更严重、更特异或发生频率高于RSI描述的个例，仍可能属于非预期；
- RSI使用MedDRA SOC/PT和可解释的分母或频率，不使用“感染、皮疹、心律失常”等过宽术语遮蔽具体风险；
- 不同适应证或人群安全性特征不同，应考虑分开呈现；
- 尚无预期严重不良反应时，仍保留独立RSI章节并明确说明。

### 3.2 RSI版本与变更

- 以事件发生时适用的RSI版本判断预期性；
- 随访报告沿用初始报告的RSI版本，不用更新版追溯性降低报告级别；
- DSUR累计汇总使用哪个RSI版本，应按E2F、CDE要求和报告周期规则预先锁定；
- RSI变更必须有医学依据、变更控制、实施日期、影响评估、版本追溯和MedDRA版本影响评估；
- 紧急安全信息不能等待年度更新，应按风险采取IB其他章节更新、方案/ICF修订、紧急安全措施和监管沟通。

### 3.3 抗肿瘤SUSAR汇总分析

不要把SUSAR处理停在个例快速报告。至少从以下维度累计分析：

1. **数量与发生率**：早期暴露量小时先关注绝对例数，暴露增加后同时关注发生率和时间趋势；
2. **严重性与转归**：优先评估死亡、危及生命、未恢复、恶化、导致停药和重要医学事件；
3. **医学特殊性**：关注罕见事件、机制相关事件、严重程度升级及可能改变获益风险的常见毒性；
4. **因果性**：综合时间关系、去激发/再激发、替代病因、疾病背景、合并用药、同类药、非临床和机制证据；
5. **类似事件聚合**：使用MedDRA层级、SMQ或预定义医学概念集，避免同一风险被不同PT拆散；
6. **联合用药**：评价单药、叠加、相互作用和联合方案特异风险，不强行只归因于一个成分。

CDE抗肿瘤指导原则提出：早期研发中同一PT或医学概念累积2例、尤其3例以上应高度关注；某些机制相关、特定医学事件或严重个例即使1例也可能触发深入评估。该内容是信号检测提示，不是所有产品、法域和场景通用的法定报告阈值。

### 3.4 信号确认后的行动

根据风险程度考虑：

- 增加访视、实验室监测、预处理、给药速度控制、停药或再挑战规则；
- 修订方案、ICF、IB/RSI、监查计划和医学监察计划；
- 重新培训中心并确认执行；
- 评估是否需要暂停入组、暂停给药、调整剂量或终止研发；
- 对可能显著影响获益风险的风险，通过适用的“其他潜在严重安全性风险信息”等路径与监管机构沟通；
- 把决定、责任人、实施日期、受影响试验和有效性检查纳入闭环。

### 3.5 ICH E2F与DSUR

E2F要求DSUR围绕同一活性成分的全球研发安全信息，连接报告周期、累计暴露、个例和汇总数据、重要发现、整体安全性评价、重要风险和结论。

DSUR的核心不是目录，而是：

`本周期新信息 → 与既往认识比较 → 风险是否变化 → 获益风险是否变化 → 已采取/拟采取行动`

#### 3.5.1 报告边界与资料包

写作前锁定：

- DIBD、DLP、报告周期和适用RSI/IB版本；
- 同一活性成分的全球试验范围、试验状态和合作方责任；
- 上一期DSUR、全球试验主清单和地区附加要求；
- 安全数据库、SAR行列表、SAE累计汇总和累计暴露；
- 临床数据库、医学审查输出、信号和风险管理记录；
- 非临床、文献、上市后数据及监管/伦理安全措施；
- 数据截止版本、对账状态和合作协议中的安全信息交换责任。

资料缺失时制作缺口表，标明责任方、截止日期和对结论的影响。不得用“未见明显风险”掩盖未收到、未编码、未随访或未对账的数据。

#### 3.5.2 内容与依赖

DSUR至少连接：

- 报告范围、全球批准和因安全原因采取的措施；
- RSI变更、试验清单和累计暴露；
- 报告期SAR行列表和累计SAE汇总；
- 临床、非干预、上市后、非临床和文献信息；
- 其他DSUR、缺乏疗效、区域信息和DLP后重要信息；
- 整体安全性评价、重要风险总结、获益风险和结论；
- 适用附录及区域补充资料。

累计暴露、试验状态、行列表、汇总表和正文口径必须一致。每个重要新信息都回答：是否改变既往认识、是否形成或改变风险、是否影响受试者或研发，以及已经或计划采取什么行动。

#### 3.5.3 DSUR质量门

提交前核对：

- DIBD、DLP、周期和版本无冲突；
- 全球试验清单与暴露、行列表和汇总表范围一致；
- RSI变更、信号、监管措施和风险控制在相关章节贯通；
- 数字、术语、MedDRA版本、产品名称和重要风险表述一致；
- 结论没有超出数据完整性和证据强度；
- 地区递交时限、附加表格和系统入口已从当前官方原文核验。

## 4. CSR与ICH E3

E3规定单项临床研究报告应让审评者理解研究为何这样设计、实际如何实施、数据如何分析、结果如何解释，以及偏离计划的事项是否改变结论。

### 4.1 E3主链

`标题与概要 → 伦理和管理结构 → 目标 → 研究计划 → 受试者处置/偏离 → 疗效 → 安全性 → 讨论和结论 → 表图 → 附录`

至少验证：

- 方案、修订案和SAP中的计划与CSR描述一致；
- 研究过程或分析计划变更有时间点、原因、盲态状态和影响评估；
- 受试者流向、分析集、偏离、停药和退出能够对账；
- 疗效结论与主要/关键次要终点、多重性和敏感性分析一致；
- 安全性按暴露、AE、死亡、SAE、其他重要事件、实验室、生命体征和临床解释形成整体结论；
- 正文、TFL、个体列表、CRF样本和附录引用可追溯。

E3英文和中文资料属于同一主题的不同语言版本，不作为两个独立证据来源。遇到翻译歧义时回到ICH当前官方原文。

### 4.2 CSR资料依赖与一致性链

正式写作前锁定：最终方案及修订、SAP及变更、数据审核和分析集决定、数据库版本和锁定/解锁记录、方案偏离、最终TFL、编码版本、病例叙述、盲态审核记录及适用附录。

逐项核对：

`最终方案/修订 → SAP及变更 → 数据审核与分析集决定 → 数据库版本 → TFL → CSR正文/概要 → 附录`

重要变更说明发生时间、理由、是否接触盲态或非盲数据，以及对偏倚和结论的潜在影响。缺关键源文件时，只输出缺口、风险和下一质量门，不宣称“可提交”。

### 4.3 CSR结果呈现

- 先说明受试者流、治疗暴露和数据可用性；
- 主要分析围绕预先规定的估计目标，报告效应量、区间和必要检验；
- 次要、探索性和事后分析明确标识；
- 亚组重点解释一致性和交互，不按单个亚组的显著性孤立下结论；
- 安全性结合暴露、时间、剂量、严重性、因果性、停药、死亡和临床背景；
- 病例叙述与表格、行列表、数据库和安全报告保持一致；
- 讨论部分回答研究能支持什么、不能支持什么，以及局限如何影响临床解释。

### 4.4 CSR常见失真

- 只写显著结果，或把无统计学显著性写成等效、无差异或无风险；
- 不解释缺失数据、伴发事件、重要偏离或分析集变化；
- 人数、版本、日期、终点定义和表格口径前后不一致；
- 正文表述超出方案、SAP或数据能够支持的范围；
- 附录缺失、版本错误或正文引用断裂；
- 把TFL逐句转写为正文，却没有医学解释和获益风险判断。

### 4.5 CSR提交级质量门

- 方案、SAP、数据库、TFL、正文、概要和附录版本已锁定；
- 受试者处置、分析集、偏离、暴露、疗效和安全性人数完成对账；
- 估计目标、主要分析、敏感性分析和结论一致；
- 死亡、SAE、重要AE、停药和病例叙述交叉一致；
- 所有重要变更、局限和数据缺口已如实解释；
- 表图、列表、附录引用、签署和批准状态满足交付要求。

## 5. GCP与ICH E6

### 5.1 版本纪律

本地E6(R1)可用于理解经典责任、知情同意、研究者/申办者职责、监查、稽查、方案、IB和必需文件框架，但不得作为当前GCP唯一依据。

回答当前GCP问题时：

1. 查ICH官网确认E6(R3)原则、附件和Step状态；
2. 查NMPA及相关部门当前中国GCP的发布、实施日和过渡安排；
3. 以事件发生或活动执行日期判断适用版本；
4. 同时检查项目方案、SOP、伦理批准和合同是否提出更具体要求。

### 5.2 不随版本变化的判断主线

- 受试者权利、安全和福祉优先于科学和社会利益；
- 研究必须具有科学合理性，风险与预期获益可接受；
- 责任可分派、任务可委托，但申办者和研究者的监督责任不能因此消失；
- 知情同意是持续过程，不是一次签字；
- 数据应当可信、可追溯、适合预定用途；
- 质量管理聚焦影响受试者保护和结果可靠性的关键因素；
- 计算机化系统、数据治理、服务提供方和新技术必须纳入风险控制和监督。

## 6. 统计与ICH E9/E9(R1)

### 6.1 ICH E9基础

E9把统计原则贯穿设计、实施、分析和报告：

- 通过随机化、盲法和预先设定减少偏倚；
- 根据假设、效应、变异、alpha、把握度、多重性、分配比和不可评价比例确定样本量；
- 明确分析集及排除规则，保持随机化优势；
- 预先规定主要分析、期中分析、停止规则、协变量、亚组和多重性控制；
- 对缺失、离群、数据转换和分析变更说明假设及影响；
- 同时报告估计值、置信区间和临床意义，不只报告P值；
- 安全性评价结合暴露、事件类型、严重程度、时间、转归和临床背景。

### 6.2 ICH E9(R1)估计目标链

按以下顺序设计：

`试验目的 → 临床问题 → 估计目标 → 主估计方法 → 敏感性估计方法 → 估计值与解释`

估计目标至少定义：

1. 治疗条件；
2. 目标人群；
3. 变量或终点；
4. 伴发事件及其策略；
5. 人群层面的汇总量。

常见伴发事件策略包括疗法策略、假想策略、复合变量策略、在治策略和主层策略。不同伴发事件可以使用不同策略，但每个选择都必须对应清楚的临床问题。

区分：

- 停止治疗、使用救援药、交叉治疗或死亡可能是伴发事件；
- 退出研究或未收集到本应收集的数据造成缺失；
- 敏感性分析针对同一估计目标下主要分析关键假设的稳健性；
- 补充分析回答不同问题，不能冒充敏感性分析。

估计目标必须回写方案，数据采集和随访必须支持它，SAP必须给出可复现的估计方法，CSR必须按相同临床问题解释结果。

## 7. CTD与ICH M4系列

### 7.1 CTD五模块

| 模块 | 核心内容 | 主要依据 |
|---|---|---|
| 模块1 | 地区行政资料、申请表、处方信息等 | 地区特异要求，不属于统一CTD主体 |
| 模块2 | CTD前言、质量/非临床/临床综述与总结 | M4、M4Q、M4S、M4E |
| 模块3 | 质量主体数据 | M4Q及当前ICH Q系列/地区要求 |
| 模块4 | 非临床研究报告 | M4S及当前ICH S系列 |
| 模块5 | 临床研究列表、CSR及相关临床报告 | M4E、E3及其他临床指导原则 |

### 7.2 M4E：临床综述、临床总结与CSR

- **2.5临床综述**：解释开发策略、证据强度和局限，形成适应证层面的获益风险判断，并说明如何支持说明书；不是各研究结果的简单重复。
- **2.7临床总结**：对生物药剂学、临床药理、有效性、安全性和单项研究进行较详实的跨研究总结和比较。
- **模块5**：存放研究列表、单项CSR、汇总分析报告、上市后报告及必要个体资料。
- **E3 CSR**：回答单项研究；**M4E 2.7**回答跨研究事实；**M4E 2.5**回答整个开发项目的监管解释和获益风险。

### 7.3 M4Q与M4S

- M4Q组织质量综述和模块3，不把生产、质量控制、稳定性、容器密封或原辅料问题塞入临床论证；涉及当前技术要求时继续检索ICH Q系列及NMPA/CDE文件。
- M4S组织非临床综述、文字/列表总结和模块4报告；非临床药理、药代和毒理结论必须能够支持临床起始剂量、风险监测、特殊人群和申报获益风险。

### 7.4 文件粒度和生命周期

- 同一信息只保留一个权威位置，其他模块交叉引用；
- 文件粒度应支持生命周期替换、版本控制和审评导航；
- 模块编号完整不等于资料充分，必须检查证据缺口和跨模块一致性；
- eCTD版本、文件技术规范和地区递交验证规则从当前官方实施文件核验。

### 8.2 中国NMPA/CDE基础路由

- 《药品管理法》及现行配套法规；
- 《药品注册管理办法》及相关实施文件；
- 当前适用的《药物临床试验质量管理规范》及过渡安排；
- 药物临床试验期间安全信息评估与管理、快速报告、DSUR管理和电子传输要求；
- 临床试验登记与信息公示、机构备案、核查/检查和数据递交要求；
- CDE按化学药、生物制品、中药、疫苗、细胞/基因治疗、儿童、罕见病、肿瘤等发布的产品和治疗领域指导原则；

### 9.2 检索流程

先把问题转成：

`法域 + 产品 + 阶段 + 主题 + 文件类型 + 活动日期`

每个候选文件核对：

- 官方完整标题和发布机构；
- 文号、版本、ICH Step或修订号；
- 正式、草案、待生效、替代、撤回或历史状态；
- 发布日与实施日；
- 适用产品、人群、阶段、角色和活动；
- 支持结论的章节、条款、表格、脚注或附件；
- 官方页面和官方附件链接；
- 检索日期。

搜索摘要只用于定位，不能代替原文。PDF表格、脚注、流程图或附件影响含义时，检查对应页面图像。不得仅凭文件名中的版本号判断当前性。

### 9.3 推荐检索词

- `文件全名 + 发布/实施/附件`；
- `site:ich.org 主题 + guideline + Step`；
- `site:cde.org.cn 产品/适应证 + 临床试验技术指导原则`；
- `site:cde.org.cn SUSAR/RSI/DSUR + 安全性`；
- `site:nmpa.gov.cn 药物临床试验质量管理规范 + 实施`；
- `site:nmpa.gov.cn 药品注册管理办法 + 临床试验`。

## 10. 冲突处理、引用和停止规则

### 10.1 冲突处理

依次比较：

1. 法域和活动日期；
2. 法律或监管层级；
3. 正式版、待生效版、草案和历史版；
4. 通用文件与产品/治疗领域文件；
5. 问题范围和适用角色；
6. 项目批准条件、方案和SOP是否提出更严格要求。

无法消解时并列说明冲突和影响，不自行宣布文件失效。

### 10.2 引用规则

法规类回答采用：`文件名（版本/日期），章节或条款 + 官方链接 + 检索日期`。

明确标记：

- `法规或强制性监管要求`；
- `正式技术指导原则建议`；
- `草案/问答/示例`；
- `方法学判断`；
- `项目实践建议`。

### 10.3 停止规则

官网不可访问、只找到二手来源、版本状态冲突、缺少正文定位或法域/日期未知且会改变结论时，停止作确定判断。输出：

- 尚未确认什么；
- 为什么无法确认；
- 影响哪些结论；
- 核验前可采取的保守措施；
- 官网、检索词、需核对字段和用户可回传的页面/PDF/截图。

## 11. 最小法规回答模板

1. **结论**：当前能够确认到什么程度；
2. **适用边界**：法域、产品、阶段、角色和活动日期；
3. **文件职责**：适用的法规/指导原则分别解决什么问题；
4. **官方依据**：文件状态、正文定位、链接和检索日期；
5. **项目影响**：对受试者、方案、IB/RSI、数据、统计、运营、DSUR/CSR或申报资料的影响；
6. **立即行动**：责任人、时间点、记录、升级和关闭证据；
7. **未核实事项**：缺什么、影响什么、下一步如何核验。
