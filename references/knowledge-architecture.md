# 内嵌知识架构

## 1. 知识组织原则

按“来源负责什么”组织知识，而不是把所有材料等权混合：

- 法律法规与实施文件：确定强制义务、责任、时限和程序；
- ICH与官方技术指导原则：确定协调化原则、方法学期望和文件结构；
- 治疗/产品特异性指导原则：确定具体人群、终点、对照或安全要求；
- 教材与行业方法：解释概念、串联流程、提供实施检查表和常见风险；
- 项目源文件与数据：决定该项目可写、可判定和可提交的事实。

## 4. 跨模块依赖

- 临床问题决定估计目标，估计目标决定终点、数据收集和分析；
- 方案决定SoA、CRF、随机、供应、安全、监查、SAP和CSR；
- 关键质量因素决定风险评估、监查策略、数据控制和CAPA优先级；
- RSI决定预期性判断、安全报告和DSUR风险变化；
- 方案、SAP、数据审核决定、数据库和TFL共同约束CSR；
- 法域与活动日期决定适用版本，不能只按文件标题判断。

## 5. 动态知识扩展

内嵌知识不是封闭边界。遇到新技术、新适应证、特殊人群或最新监管问题时，按 `guideline-catalog.md → official-retrieval.md → 主题模块` 扩展证据。只在当前回答中使用检索结果；除非用户明确要求更新Skill，不自动把临时网页内容写回内嵌知识。
